<h1 id="ConfigurationManagementStrategy">Configuration Management Strategy</h1>
<h2 id="Thebasics">The basics</h2>
<ol>
    <li>All new features will be added on the trunk.</li>
    <li>A branch will created for each minor release.</li>
    <li>All bugfixes will be made on the relevant branch.</li>
</ol>
<h2 id="Tagging">Tagging</h2>
<ol>
    <li>A tag will be created:
    <ol>
        <li>at the end of each iteration (tags will be named 'iteration-n' where n is the iteration number)</li>
        <li>for each release (tags will be named 'release-version' where version is the version number in major.minor.bugfix format).</li>
    </ol>
    </li>
</ol>
<h2 id="Branches">Branches</h2>
<ol>
    <li>A branch will be created for each minor  release (7.0, 7.1, 7.2, etc.)</li>
    <li>Branches will be named 'maintenance-major.minor'.</li>
</ol>
<h2 id="Makingabugfix">Making a bugfix</h2>
<ol>
    <li>Should always happen in a branch.</li>
    <li>What is our strategy for disseminating the fix to
    <ol>
        <li>Other branches (i.e. other versions)?</li>
        <li>The trunk?</li>
    </ol>
    </li>
</ol>
